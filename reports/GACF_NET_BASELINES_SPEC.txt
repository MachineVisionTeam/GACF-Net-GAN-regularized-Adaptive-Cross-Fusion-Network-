------------------------------------------------------------
1.2  Why it is the right baseline for this work
------------------------------------------------------------

  - AGAFNet (2026) is the most recent state-of-the-art on the exact same
    three datasets we evaluate on (Lizard, CoNSeP, PanNuke).
  - Joint segmentation + per-nucleus classification — same task formulation.
  - Uses the same evaluation protocol we adopt (Convention C per-class F1
    under IoU >= 0.5 instance matching).
  - Authors do NOT release official code or trained weights; we therefore
    re-implement the architecture exactly per the paper's recipe (see
    GACF_NET_ARCHITECTURE_SPEC Section 8.1) and report a documented
    replication gap.

------------------------------------------------------------
1.3  Two comparisons we run against AGAFNet
------------------------------------------------------------

(a) END-TO-END comparison
    AGAFNet's full pipeline (segmentation + classification) vs GACF-Net's
    full pipeline (Stage 1 segmentation + Stage 2 fusion + XGBoost).
    Reported as standard per-dataset segmentation metrics (ACC, Dice, SE,
    AJI, PQ) plus Convention C per-class F1.
    -> Shows whether the WHOLE GACF-Net pipeline beats AGAFNet end-to-end.

(b) CONTROLLED HEAD-TO-HEAD (the +graph+cross column)
    Hold AGAFNet's segmentation FIXED at its predicted instance masks and
    swap ONLY the classifier head:
      - AGAFNet-MV   : AGAFNet's own pixel-majority-vote per nucleus
      - GACF-Net S2  : XGBoost on Stage-2 fusion embeddings (+graph+cross
                       mode) over the SAME AGAFNet masks
    Evaluated under Convention C with IDENTICAL FN_d for both methods
    (since the segmentation is fixed). Isolates classifier quality from
    segmentation quality.
    -> Shows whether OUR Stage-2 classifier beats AGAFNet's pixel-MV
       classifier on the same nuclei.

(See GACF_NET_ARCHITECTURE_SPEC Section 8 for the full experimental
design; numerical results are in GACF_NET_FINAL_RESULTS.txt.)

4. HOW TO PHRASE THE COMPARISON IN THE PAPER
================================================================================

Suggested two-paragraph framing for the Results section (Framing A —
pipeline-first, as decided in the architecture-spec contributions section):

Paragraph 1 — end-to-end:
  "We compare GACF-Net end-to-end against AGAFNet [Naing2026] on three
   public H&E datasets (Lizard, CoNSeP, PanNuke). For AGAFNet, we re-
   implement the architecture per the paper's recipe and report a
   documented replication gap; we adopt the same five-fold CV protocol
   for Lizard / CoNSeP and the standard three-fold CV for PanNuke.
   Table X reports the per-dataset segmentation and classification
   results."

Paragraph 2 — controlled head-to-head (Section 8.2 protocol):
  "To isolate the classifier contribution from the segmentation
   contribution, we run a controlled head-to-head experiment in which we
   fix the predicted instance masks to AGAFNet's output and swap only the
   per-nucleus classifier. We compare AGAFNet's native pixel-majority-vote
   classifier against our Stage-2 fusion + XGBoost head, both evaluated
   under Convention C with identical FN_d. Table Y reports the resulting
   per-class F1 — our classifier matches AGAFNet on Lizard (strong
   segmentation) and improves significantly on CoNSeP and PanNuke (weaker
   segmentation)."

(Numerical values for Table X and Table Y live in GACF_NET_FINAL_RESULTS.txt.)
